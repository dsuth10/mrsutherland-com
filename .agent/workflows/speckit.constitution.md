# /speckit.constitution

Use this command to create or update the project's constitution (`.specify/rules/constitution.md`).

## Workflow

1.  **Analyze Context**: Review project specifications and any existing rules.
2.  **Socratic Review**: Ask about architectural preferences, coding standards, and project-specific constraints.
3.  **Generate Constitution**: Create or update `.specify/rules/constitution.md` with:
    - Core Principles (e.g., local-first, privacy-focused).
    - Tech Stack constraints.
    - Coding Guidelines (standard formatting, testing requirements).
    - Project-specific rules (e.g., "no-spellcheck" editor specifics).
